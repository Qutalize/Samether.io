# サメザリオ (Samezario) 仕様書

## 1. 概要

Slither.ioにインスパイアされた位置情報連動型リアルタイムマルチプレイヤーゲーム。
プレイヤーは現実世界を歩き回りながらサメを操作し、海中でエサ（小魚）を食べて成長・進化しながら他プレイヤーと競う。
対象ユーザー：全年齢・性別
いつ・どんな場面で：どこでも暇なとき、勝手に健康になれる（通信環境に左右されない）

### 挑戦したいこと
1.大人数の接続に耐えるトラフィック処理
2.WebSocketでデータのリアルタイム性の担保
3.DBとキャッシュのカラムのチューニングと使い分け
4.水の質感、えさの波による浮遊感、サメの種類に応じた見た目の表現
5.領域制度の確立（めっちゃ重そう）
6.GPSデータの処理

## 2. 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Phaser + HTML5 Canvas + TypeScript（モバイルブラウザ対応） |
| バックエンド | Go + WebSocket (ws) |
| ビルドツール | Vite or Amplify |
| 位置情報 | Amazon Location Service（移動距離の算出・トラッキング） |
| インフラ | AWS (EC2 or ECS + ALB + ElastiCache) |
| DB | 最小構成（暫定1位のみ保持。Redis or インメモリ） |
| パッケージマネージャ | npm |

### インフラ方針
- リアルタイム通信が主要ボトルネックのため、WebSocket のスケーリングを最優先
- 負荷が問題になった場合 ALB で WebSocket サーバーを水平スケール
- Terraform は初期段階では不要。手動構築 → 必要に応じて IaC 化

## 3. 操作システム

### 3.1 GPS モード（屋外）

- スマートフォンの GPS で現実世界の位置を取得
- 現実の移動がゲームマップ上のサメの移動に反映
- **移動距離がチャージポイント (CP) として蓄積される**（後述）
- 移動距離の算出には **Amazon Location Service** を使用
  - Tracker を用いてデバイスの位置を継続的にトラッキング
  - サーバー側で位置履歴を取得し、移動距離 (メートル) を算出して CP に変換
  - 参考: https://aws.amazon.com/jp/location/

### 3.2 タッチ操作モード（室内）

- 画面上の指操作（スワイプ / バーチャルジョイスティック）でもサメを動かせる
- サメの前方に **扇形のレーダー** が点滅表示
- レーダー範囲内のエサ・他のサメを一時的に可視化
- レーダーの点滅周期はサメによって変わる

## 4. チャージポイント (CP) システム

現実世界での移動距離がゲーム内リソースに変換される仕組み。

| 項目 | 内容 |
|---|---|
| 蓄積条件 | GPS モードでの移動距離 (メートル単位) |
| 蓄積レート | 50m 移動 = 1 CP |
| 用途 | ダッシュ、特殊能力の発動 |
| ダッシュ消費 | CP × サメ種別のスケール係数 で移動距離が決まる |

- 攻撃系のサメ（ホオジロザメ等）は CP 消費効率が高い（少ない CP で長く走れる）が、**時間制限があるため大量の CP が必要** → 外で沢山歩く必要がある
- 非攻撃系・深海魚系のサメは CP 効率は低いが、必要量も少ない
- ゲームプレイ中でも移動しながら CP を蓄積可能
- CPの貯め方は，スタート地点とゴール地点だけ記録しておけば，AWSの機能から移動距離を算出できる

## 5. サメの進化システム

### 5.1 進化ツリー概要

ゲーム開始時に **3つのルート** から1つを選択し、各ルートの1段目のサメからスタートする。
全ルート共通で **5段階進化**。一定スコアに達すると次の段階に進化できる。

```
           攻撃系             非攻撃系            深海魚系
          ─────             ────────           ──────
  1段目  シュモクザメ         ドチザメ          ツラナガコビトザメ
            |                  |                    |
  2段目  イタチザメ          ネムリブカ           ノコギリザメ
            |                  |                    |
  3段目  アオザメ(最速)      シロワニ(燃費最高)     ラブカ
            |                  |                    |
  4段目  ホオジロザメ         ウバザメ          ゴブリンシャーク
            |                  |               (ミツクリザメ)
  5段目  メガロドン          ジンベエザメ        ニシオンデンザメ
        (攻撃系最上位)      (非攻撃系最上位)     (深海魚系最上位)
```

**スタート種の背景:**
- **シュモクザメ**: サメ界で最も個体数が多く、ヒエラルキーが低い。一度に10頭ほど生まれる
- **ドチザメ**: 小型肉食サメ。温和で非攻撃系の入口にふさわしい
- **ツラナガコビトザメ**: 小型の深海魚。弱いがロマン型ルートの出発点

### 5.2 攻撃系ルート

速度と攻撃力に特化。ただしエサの消費が激しく、時間制限がある。

| 種 | 段 | 特徴 |
|---|---|---|
| シュモクザメ | 1 | 攻撃系の入口。個体数が多くヒエラルキーが低い |
| イタチザメ | 2 | バランス型。速度やや速い |
| アオザメ | 3 | **全サメ最速**。燃費が悪い |
| ホオジロザメ | 4 | 大型。噛みちぎりスタイル（他サメの体節を数個奪える） |
| メガロドン | 5 | 攻撃系最上位。巨大・高速・強力。**短命**（一定秒ごとにエサが必要、なければ退化）。**血の知覚**: 他サメの方向がわかる（レーダー不要） |

**攻撃系の食事スタイル**: 噛みちぎり — 他サメの体節に頭をぶつけると、相手を殺さずに **体節を数個奪い取る**（ホオジロザメ以上）

### 5.3 非攻撃系ルート

耐久力と効率に特化。丸呑みスタイルでエサを大量摂取。

| 種 | 段 | 特徴 |
|---|---|---|
| ドチザメ | 1 | 小型肉食。非攻撃系の入口 |
| ネムリブカ | 2 | やや遅いが燃費が良い |
| シロワニ | 3 | **超燃費型**（少量のエサで長時間維持可能。1日サバ1匹で満足） |
| ウバザメ | 4 | 大型。ジンベエザメ寄りの性能 |
| ジンベエザメ | 5 | 非攻撃系最上位。巨大。**丸呑み**: エサを広範囲で一括取得。オーソドックスで安定した性能 |

**非攻撃系の食事スタイル**: 丸呑み — 口の前方の広い範囲でエサを一括吸収

### 5.4 深海魚系ルート（ロマン型）

進化途中は弱いが、最上位のニシオンデンザメが非常に強力。

| 種 | 段 | 特徴 |
|---|---|---|
| ツラナガコビトザメ | 1 | 小型で弱い。探索範囲がやや広い |
| ノコギリザメ | 2 | ノコギリで近接攻撃可能 |
| ラブカ | 3 | 深海適応。探索範囲が広がる |
| ゴブリンシャーク (ミツクリザメ) | 4 | 射程の長い噛みつき攻撃 |
| ニシオンデンザメ | 5 | 深海魚系最上位。**探索範囲最大**、**エサをあまり必要としない**、**エサを食べなくても自動成長**、スピードは遅い。現実のニシオンデンザメは400歳の個体もいる最長老 |

**特殊アイテム**: マップ上に稀に出現する特殊アイテムを取得すると、即座にニシオンデンザメに変身できる（種族関係なし）

### 5.5 特殊種

| 種 | 備考 |
|---|---|
| ネコザメ | 特殊なエサ「貝」のみ食べられる。ニッチなプレイスタイル |

※ コバンザメはサメではないため登場しない

### 5.6 サメの衝突ルール

- 通常の Slither.io ルール適用 — **鼻（頭）が相手の体節に当たった方が死亡**
- 死んだら，イベント(ポイントが放出 or エフェクト発生)が起こる

## 6. 領域システム（後回し．なんかむずそう）

### 6.1 領域の形成

- サメが移動すると軌跡が残る
- 軌跡が自己交差した際の内側領域をそのサメの領域とする
- 領域は他のサメの移動や領域確立に伴い動的に変化する

### 6.2 領域の強さ

- 領域の強さ = そのサメの種族ランク（進化段階）
- **弱いサメ** が **強いサメの領域** に入ると → **即死（確キル）**
- **強いサメ** が **弱いサメの領域** に入ると → **スルー（無影響）**
- 同格サメの領域 → 通常の衝突ルールが適用

### 6.3 バランス調整（強者抑制メカニズム）：これは必須

強いサメほど領域が拡大しやすい収束問題への対策:

| 対策 | 内容 |
|---|---|
| 速度バランス | 弱いサメほど基本移動速度が速い（逃げやすい） |
| 燃費バランス | 強い攻撃系サメほどエサの消費が激しい（維持コストが高い） |
| 領域減衰 | 軌跡は時間経過で消える（古い部分から消失） |
| サメの生態準拠 | スピードが速いほど多くのエサが必要（現実のサメの生態と一致） |

## 7. 衝突・戦闘ルール

### 7.1 基本ルール（Slither.io 準拠）

- 自分の **鼻（頭の先端）** が他のサメの **体節** に衝突 → **自分が死亡**
- サメの鼻は弱点（現実のサメの生態と一致）
- 自分自身の体節には衝突しない

### 7.2 階級差がある場合

- 上位種が下位種の体節にぶつかっても死ぬ（鼻は弱点）
  - 例: ホオジロザメの鼻がドチザメの腹に当たったら、ホオジロザメが死ぬ
- ただし上位攻撃系サメ（ホオジロザメ以上）は「噛みちぎり」で体節を奪える特殊判定あり（あったら面白いけど，衝突との違いがわからない and 噛みちぎりのメリットがないかも．→一旦なし）

### 7.3 死亡時の処理

- サメの全体節がエサに変換され、死亡地点周辺に散らばる
- プレイヤーはリスポーン画面に戻り、シュモクザメ(そのルートの最初のサメ)から再スタート

## 8. エサシステム

### 8.1 エサの種類（最初は１種類，光っているものが餌にする．）

| エサ | 出現条件 | 効果 |
|---|---|---|
| 小魚 (S) | マップ上に自然発生 | +1 経験値 |
| 小魚群 (M) | マップ上にやや稀に自然発生 | +3 経験値 |
| 残骸 (L) | サメ死亡時に散らばる | +5 経験値 |
| 貝 | マップ上に稀に発生 | ネコザメ専用エサ |
| 特殊アイテム | マップ上に極稀に発生 | ニシオンデンザメ化 |

### 8.2 食事スタイル（一旦なし．通常捕食のみ）

| スタイル | 対象種 | 挙動 |
|---|---|---|
| 通常捕食 | シュモクザメ等 | 頭がエサに接触で取得 |
| 噛みちぎり | ホオジロザメ以上の攻撃系 | 他サメの体節を奪い取れる |
| 丸呑み | ジンベエザメ等の非攻撃系上位 | 前方広範囲のエサを一括吸収 |

- Slither.io の虹色に動くオブジェクト = 小魚のイメージ：動かすと重くなりそうだから，一旦静的オブジェクト

## 12. ボットシステム（一旦なし）

- ユーザー数が少ない時のゲーム体験を維持するため、サーバーサイドでボットを投入
- ボットの行動パターン:
  - エサに向かって移動
  - 近くの弱いサメを追いかける
  - 強いサメから逃げる
- ボットも進化する（時間経過でランダムに進化先を選択）

## 13. インフラ構成

Amazon GameLift を使うと良いかも？：上記で「一旦なし」にした高負荷な部分を担ってくれる：ブラウザならサブミッション的にした方が良さそう

```
[モバイルブラウザ]
       |
       ├── WebSocket (wss://)
       |         |
       |     [ALB] ← 負荷が問題になった場合に導入
       |         |
       |     [Fargate / ECS]
       |     ├── WebSocket サーバー (Node.js)
       |     ├── ゲームロジック
       |     └── ボット管理
       |         |
       |     [ElastiCache (Redis)] ← リーダーボード・セッション管理（最小構成）
       |
       └── GPS 位置情報
                 |
           [Amazon Location Service]
           ├── Tracker (デバイス位置のトラッキング)
           └── 移動距離算出 → CP 変換
```

### スケーリング方針
1. 初期: 単一 Fargate タスクで稼働（処理の強いコアを選ぶ）
2. 負荷が問題になった場合: オートスケーリング + ALB + 複数タスク：オートスケーリングは20秒くらいかかりそうでゲームサーバが落ちそう
3. Redis はリーダーボード（暫定1位のみ）とセッション管理に使用
4. DB は基本的に不要（ランキングは暫定1位のみ保持）

## 付録 A: サメ雑学メモ

- サメは約 553 種類（2020年時点）
- シュモクザメは一度に約10頭生まれる（他のサメは通常2頭）
- ニシオンデンザメは最長寿のサメで、約400歳の個体が確認されている
- アオザメは全サメの中で最速
- シロワニは1日にサバ1匹で満足するほど燃費が良い
- コバンザメはサメではない
- 茨城の大洗水族館にはサメの飼育種類数が世界一（とされる）
- サメのほとんどは深海魚
